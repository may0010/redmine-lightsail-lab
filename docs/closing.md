# クリーンアップ手順

このドキュメントでは、AWS Lightsail 上に構築した Redmine 検証環境を削除する手順を説明します。

## 前提条件

環境を削除すると、インスタンス内のデータは失われます。

必要なデータがある場合は、事前にバックアップまたはスナップショットを取得してください。

---

## 1. Lightsail インスタンスの削除

1. AWS マネジメントコンソールを開きます。
2. **Lightsail** を開きます。
3. 左メニューの **「インスタンス」** を選択します。
4. 削除するインスタンスの **︙（アクション）** をクリックします。
5. **「削除」** を選択します。
6. 確認画面の内容を確認し、インスタンスを削除します。

削除が完了すると、インスタンスに対する課金は停止します。

---

## 2. Static IP の解放

> **注意**
>
> Static IP を作成していない場合、この手順は不要です。

1. Lightsail の左メニューから **「ネットワーク」** を選択します。
2. **Static IP** の一覧を表示します。
3. 解放する Static IP の **︙（アクション）** をクリックします。
4. **「Static IP を解放（Release static IP）」** を選択します。
5. 確認画面で内容を確認し、解放します。

不要な Static IP を残していると、インスタンス削除後も課金対象となる場合があります。

---

## 3. 削除確認

以下を確認します。

* インスタンス一覧に対象のインスタンスが表示されないこと
* Static IP 一覧に不要な Static IP が表示されないこと

---

## チェックリスト

環境の削除が完了していることを確認してください。

* ☐ Lightsail インスタンスを削除した
* ☐ 不要な Static IP を解放した
* ☐ 必要なデータのバックアップを取得した（必要な場合）

これで、個人検証用に構築した Redmine 環境のクリーンアップは完了です。
